Getting Started

Learn how to create and implement .cursorrules files in your Cursor IDE projects. Follow this step-by-step guide to get up and running quickly.

Prerequisites

Before you begin, make sure you have the following:

  • Cursor IDE - Download and install from cursor.sh
  • Git - For version control (optional but recommended)
  • Basic understanding of your project's technology stack
Note: .cursorrules files work with any project type - web development, mobile apps, backend services, data science, and more!

Step 1: Create Your .cursorrules File

Navigate to Your Project Root

Open your project in Cursor IDE and navigate to the root directory (where your main configuration files like package.json, requirements.txt, or similar are located).

Create the .cursorrules File

Create a new file named exactly .cursorrules (with the dot at the beginning). This is important - the filename must be exactly as shown.

# Using terminal/command line touch .cursorrules # Or create it directly in Cursor IDE # File → New File → Save as .cursorrules

Verify File Creation

Make sure the file is created in the correct location. You should see it in your project root alongside other configuration files.

Step 2: Understand the Basic Structure

.cursorrules files use a simple text format. Here's the basic structure you should follow:

# Role Definition You are an expert [technology/framework] developer... ## Project Context This project is a [description of your project]... ## Coding Standards - Use [specific coding style] - Follow [framework conventions] - Implement [testing requirements] ## File Structure - Components go in [directory] - Utils go in [directory] - Tests go in [directory] ## Additional Rules [Any specific requirements or preferences]
Pro Tip: Start simple and gradually add more specific rules as you learn what works best for your project.

Step 3: Write Your First Rules

Let's create a basic .cursorrules file for a React project. Replace the content with your specific needs:

# React Developer Assistant You are an expert React developer specializing in modern web applications. ## Project Context This is a React application built with TypeScript, using modern hooks and functional components. ## Coding Standards - Use TypeScript for all new files - Prefer functional components over class components - Use React hooks (useState, useEffect, etc.) - Implement proper error handling with try-catch blocks - Write comprehensive tests using Jest and React Testing Library ## File Structure - Components go in src/components/ - Hooks go in src/hooks/ - Utils go in src/utils/ - Types go in src/types/ - Tests go in __tests__/ directory ## Code Style - Use Prettier for code formatting - Follow ESLint rules - Use meaningful variable and function names - Add JSDoc comments for complex functions - Use const/let instead of var ## Additional Rules - Always include proper TypeScript types - Use React.memo for performance optimization when needed - Implement proper loading and error states - Follow accessibility best practices

Step 4: Test Your Rules

Save the File

Save your .cursorrules file and make sure it's in your project root directory.

Restart Cursor IDE

Close and reopen Cursor IDE to ensure the new rules are loaded properly.

Test with AI Assistant

Try asking the AI assistant to help you with a coding task. You should notice that the responses now follow your specified rules and conventions.

Example Test: Ask the AI to "Create a new React component for a user profile card" and see if it follows your TypeScript and file structure rules.

Step 5: Add to Version Control

It's important to include your .cursorrules file in version control so your team can share the same AI behavior:

# Add to git git add .cursorrules # Commit with descriptive message git commit -m "Add .cursorrules configuration for AI assistant" # Push to remote repository git push origin main
Team Collaboration: Make sure all team members pull the latest .cursorrules file to maintain consistency across the project.

  • Add More Specificity: Include more detailed instructions for complex scenarios
  • Explore Examples: Check out our examples page for inspiration

Common Issues

If you're having trouble, here are some common issues and solutions:

File Not Being Recognized

  • Make sure the filename is exactly .cursorrules (with the dot)
  • Ensure the file is in your project root directory
  • Try restarting Cursor IDE

Rules Not Working as Expected

  • Check your syntax - use clear, simple language
  • Make sure rules are specific enough
  • Test with simple requests first
