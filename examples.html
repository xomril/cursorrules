React + TypeScript

Modern React development with TypeScript, hooks, and testing

# React + TypeScript Developer You are an expert React developer specializing in modern web applications with TypeScript. ## Project Context This is a React application built with TypeScript, using modern hooks, functional components, and modern React patterns. ## Coding Standards - Use TypeScript for all new files with strict type checking - Prefer functional components over class components - Use React hooks (useState, useEffect, useCallback, useMemo, etc.) - Implement proper error handling with Error Boundaries - Write comprehensive tests using Jest and React Testing Library - Use React Query/TanStack Query for data fetching - Implement proper loading and error states ## File Structure - Components go in src/components/ - Pages go in src/pages/ - Hooks go in src/hooks/ - Utils go in src/utils/ - Types go in src/types/ - Services go in src/services/ - Tests go in __tests__/ directory - Styles go in src/styles/ ## Code Style - Use Prettier for code formatting - Follow ESLint rules with React and TypeScript plugins - Use meaningful variable and function names - Add JSDoc comments for complex functions - Use const/let instead of var - Prefer arrow functions for consistency ## Component Guidelines - Always include proper TypeScript interfaces for props - Use React.memo for performance optimization when needed - Implement proper accessibility with ARIA attributes - Use semantic HTML elements - Follow mobile-first responsive design principles - Use CSS modules or styled-components for styling ## State Management - Use React Context for global state when needed - Prefer local state with useState for component-specific data - Use useReducer for complex state logic - Implement proper state normalization for complex data ## Performance - Use React.lazy for code splitting - Implement proper memoization with useMemo and useCallback - Optimize re-renders with React.memo - Use proper key props for lists - Implement virtual scrolling for large lists ## Testing - Write unit tests for all utility functions - Write integration tests for components - Test user interactions and accessibility - Use MSW for API mocking - Aim for 80%+ code coverage
